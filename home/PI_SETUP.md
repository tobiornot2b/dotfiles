# Pi.dev + Nix Integration Guide

Comprehensive guide to using pi.dev with your Nix-based dotfiles configuration.

## Overview

Pi is a minimal, extensible terminal coding agent harness. This setup integrates pi.dev into your multi-host dotfiles infrastructure, supporting:

- **NixOS (Dell Precision 5560)** - Hyprland, Wayland, full system control
- **Ubuntu (Standalone)** - Home-manager only, portability
- **Shared extensions & skills** - Version-controlled, reusable across systems

## Architecture

```
~/.dotfiles/
├── home/
│   ├── pi-dev.nix                    # Main pi.dev home-manager module
│   ├── pi-extensions/                # Custom TypeScript extensions
│   │   ├── package.json
│   │   ├── tsconfig.json
│   │   ├── src/
│   │   │   ├── custom-compaction.ts  # Topic-aware message summarization
│   │   │   ├── permission-gate.ts    # Approval gates for sensitive ops
│   │   │   └── ssh-executor.ts       # Remote command execution
│   │   └── README.md
│   └── pi-skills/                    # Markdown-based skills
│       ├── nix-query.md
│       ├── nix-module-gen.md
│       ├── flake-update.md
│       ├── workspace-ops.md
│       ├── git-helpers.md
│       └── README.md
│
├── hosts/
│   ├── dell-precision-5560/home.nix  # Enable pi-dev with NixOS config
│   └── ubuntu/home.nix               # Enable pi-dev with Ubuntu config
│
└── flake.nix                          # Unchanged - pi in nixpkgs
```

## Quick Start

### Installation

The pi module is automatically enabled on both NixOS and Ubuntu hosts. To verify:

```bash
# On NixOS (Dell)
sudo nixos-rebuild switch --flake ~/.dotfiles#tobixx

# On Ubuntu
cd ~/.dotfiles
nix run .#homeConfigurations.dwp7953.activationPackage
```

### First Run

```bash
# Start pi interactively
pi

# Authenticate (choose one method)
# Option 1: OAuth subscription login
/login

# Option 2: Set API key environment variable (handled per-system)
export ANTHROPIC_API_KEY="sk-ant-..."
pi
```

### Configuration

Pi automatically loads:

1. **Global config**: `~/.pi/agent/settings.json` (generated by Nix module)
2. **Project instructions**: `.pi/AGENTS.md` in project directory
3. **Shared extensions**: `~/.pi/agent/extensions/` (created by Nix module)
4. **Shared skills**: `~/.pi/agent/skills/` (created by Nix module)
5. **Project extensions**: `.pi/extensions/` in project directory

## Module Options

The pi.dev module is configured in `home/pi-dev.nix` with these options:

```nix
home.pi-dev = {
  # Enable pi.dev (default: false)
  enable = true;
  
  # Node.js major version (default: "24")
  nodeVersion = "24";
  
  # Theme: "dark", "light", "auto" (default: "dark")
  theme = "dark";
  
  # LLM thinking level: "low", "medium", "high" (default: "medium")
  thinkingLevel = "medium";
  
  # Default model providers (default: [])
  globalModels = [ "anthropic/claude-opus" ];
  
  # Custom extensions path (default: ~/.pi/agent/extensions)
  extensionsPath = "${config.home.homeDirectory}/.pi/agent/extensions";
  
  # Custom skills path (default: ~/.pi/agent/skills)
  skillsPath = "${config.home.homeDirectory}/.pi/agent/skills";
  
  # Pi packages to install on all systems
  piPackages = [
    "npm:@juicesharp/rpiv-ask-user-question"
  ];
};
```

## Extensions

Custom TypeScript extensions live in `home/pi-extensions/src/`:

### Building Extensions

```bash
cd ~/.dotfiles/home/pi-extensions
npm install
npm run build
# Output: dist/*.js
```

### Available Extensions

#### 1. custom-compaction.ts
Topic-aware message compaction for long conversations. Intelligently groups related messages and creates semantic summaries instead of naive truncation.

**Usage**: Automatically invoked by pi when approaching context limits.

#### 2. permission-gate.ts
Adds approval gates to sensitive operations:
- Destructive file operations (rm, delete, unlink)
- Privileged commands (sudo, chmod, chown)
- Configuration file modifications

**Commands**:
```
/trust list                    # Show enabled gates
/trust enable <gate-name>      # Enable gate
/trust disable <gate-name>     # Disable gate
```

#### 3. ssh-executor.ts
Executes commands on remote systems via SSH. Useful for distributed development and remote infrastructure management.

**Setup**:
```
/ssh-config add <alias> <user@host[:port]>
/ssh-config list
/ssh-config remove <alias>
```

**Usage**:
```
/ssh-exec host:<alias> command:"ls -la"
```

## Pi Packages

Declare pi packages to install across all systems via the `piPackages` option in your home configuration.

### Configuration

Add packages to your home configuration in `hosts/*/home.nix`:

```nix
home.pi-dev = {
  enable = true;
  piPackages = [
    "npm:@juicesharp/rpiv-ask-user-question"  # Ask user questions with rich UI
    "npm:@another/pi-package@1.0.0"           # Or pin specific versions
  ];
};
```

### Installation

After updating your home configuration and running home-manager activation, install the packages with:

```bash
# Install all declared packages
for pkg in npm:@juicesharp/rpiv-ask-user-question; do
  pi install $pkg
done

# Or install individually
pi install npm:@juicesharp/rpiv-ask-user-question
```

Packages are stored in `~/.pi/agent/npm/` and are available across all systems.

### Available Packages

#### @juicesharp/rpiv-ask-user-question

Enables interactive Q&A within pi conversations. Provides rich terminal UI for asking users multiple choice or text questions.

**Features:**
- Multiple choice questions with keyboard navigation
- Text input validation
- Optional default values
- Rich text formatting in terminal UI

**Installation:**
```bash
pi install npm:@juicesharp/rpiv-ask-user-question
```

**Verify installation:**
```bash
pi list
# Should show: npm:@juicesharp/rpiv-ask-user-question
```

**Use in prompts:**
The package extends pi's UI capabilities. Use in your prompts or project AGENTS.md to ask users questions interactively.

## Skills

Markdown-based CLI tools that encapsulate domain knowledge. Automatically discovered and loaded by pi.

### Using Skills

```bash
# Query nixpkgs for a package
/nix-query nodejs

# Generate a new home-manager module
/nix-module-gen --path home/mytool --type single

# Safely update flake inputs
/flake-update --check
/flake-update --input nixpkgs --dry-run

# Manage workspace
/workspace-ops clone https://github.com/user/project
/workspace-ops init-project my-app --template nix-flake

# Git workflow
/git-commit --type feat --scope pi-dev
/git-branch --create feature/new-extension
/git-sync --all-repos
```

### Available Skills

1. **nix-query.md** - Package and version information from nixpkgs
2. **nix-module-gen.md** - Scaffold new home-manager/NixOS modules
3. **flake-update.md** - Safe flake input updates with validation
4. **workspace-ops.md** - Repository and project management
5. **git-helpers.md** - Semantic commits and multi-repo operations

See `home/pi-skills/README.md` for details.

## Authentication

Pi authentication is **per-system** and not managed by the Nix module.

### NixOS Hosts

Set environment variables before running pi:

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENAI_API_KEY="sk-..."
pi
```

Or use OAuth:
```bash
pi
/login
# Select provider from web UI
```

### Ubuntu

Same approach - set env vars or use `/login` for OAuth.

### Persistent Setup (Optional)

Create `~/.config/pi/auth.json` (after first login):
```json
{
  "providers": {
    "anthropic": { "apiKey": "sk-ant-..." },
    "openai": { "apiKey": "sk-..." }
  }
}
```

**Note**: Secrets are stored per-system and not in version control.

## Workflow Examples

### Adding a New Extension

```bash
# 1. Create TypeScript file
cat > home/pi-extensions/src/my-extension.ts << 'EOF'
export default function (pi: any) {
  pi.registerCommand({
    name: "my-cmd",
    execute: async (args, ctx) => {
      return "Hello from my extension!";
    }
  });
}
EOF

# 2. Build
cd home/pi-extensions && npm run build

# 3. Pi auto-loads from ~/.config/pi/extensions/

# 4. Test
pi
/my-cmd
```

### Working with Dotfiles

```bash
# 1. Start pi in dotfiles directory
cd ~/.dotfiles
pi "Analyze the home-manager module structure"

# 2. Use skills
/nix-query systemd
/nix-module-gen --path home/newfeature --type directory

# 3. Create semantic commit
/git-commit --type feat --scope hyprland
# Type message, pi creates: feat(hyprland): your message

# 4. Rebuild systems
sudo nixos-rebuild switch --flake .#tobixx
nix run .#homeConfigurations.dwp7953.activationPackage
```

### Distributed Development

```bash
# 1. Configure SSH hosts
pi
/ssh-config add prod "deploy@prod.example.com:22"
/ssh-config add staging "deploy@staging.example.com"
/ssh-config list

# 2. Execute remote commands
/ssh-exec host:prod command:"systemctl status myapp"
/ssh-exec host:staging command:"docker ps"

# 3. Use permission gates for safety
/trust enable privileged  # Require approval for sudo commands
```

## Troubleshooting

### Pi Command Not Found

Ensure Node.js and pi-coding-agent are installed:

```bash
# Check Node.js
node --version

# Verify pi install
which pi
# If not found, manually install:
npm install -g @earendil-works/pi-coding-agent
```

### Extensions Not Loading

Check extension paths:

```bash
# Extensions should be at:
~/.pi/agent/extensions/

# Verify symlink to dotfiles
ls -la ~/.pi/agent/extensions
# Should show dotfiles extensions linked here
```

### Settings.json Not Applied

Regenerate Nix config:

```bash
# Ubuntu
nix run .#homeConfigurations.dwp7953.activationPackage

# NixOS
sudo nixos-rebuild switch --flake .#tobixx
```

### Skills Not Appearing

Verify skills directory:

```bash
ls ~/.local/share/pi/skills/
# Should contain: nix-query.md, nix-module-gen.md, etc.

# Symlink check
ls -la ~/.local/share/pi/skills
# Should show: ... ~/.dotfiles/home/pi-skills
```

## Integration with System Tools

Pi integrates seamlessly with your existing setup:

- **Git**: `/git-commit` uses conventions from your `.dotfiles` repo
- **Nix**: `/nix-query` searches your nixpkgs input, `/flake-update` respects flake.lock
- **SSH**: `/ssh-exec` uses your `~/.ssh/config` and keys
- **Shell**: Pi inherits your shell environment, aliases, functions

## Notes

- Pi does NOT require system-level authentication setup (handled per-use)
- Extensions are TypeScript; skills are Markdown for flexibility
- All configuration is version-controlled in ~/.dotfiles
- Multi-system support: changes sync via git pull/push
- Nix ensures reproducible Node.js environment across hosts
- **Important**: Pi uses `~/.pi/agent` by default (not `~/.config/pi`). This is controlled by the `PI_CODING_AGENT_DIR` environment variable if you need to override it
